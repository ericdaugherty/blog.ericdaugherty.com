Matt Cutts has a <a href="http://www.mattcutts.com/blog/not-trapping-users-data-good/">great post</a> about how Google doesn't trap a user's data.<br /><br />I think this is part of what makes Google so attractive.  If you contrast this post with the ongoing tussle over office document formats (ODS versus Microsoft's Open XML).  Part of Microsoft Office's dominance has been their ownership of the default format.<br /><br />Microsoft isn't the only (or even worst) offender here.  I've been <a href="http://www.ericdaugherty.com/blog/2007/03/photo-managment.html">recently frustrated</a> by the closed data system of Adobe Elements, and the lack of a document tagging/category system in general.   <a href="http://blog.jonudell.net/2007/02/20/whos-got-the-tag-database-truth-versus-file-truth-part-3/">Vista has gotten some heat</a> (I think Jon is correct that there isn't an obvious right/wrong) about how Vista has 'corrupted' some image files by writing tag data into their meta data.<br /><br />Apple is another offender.  My ongoing effort to maintain the ability <a href="http://www.ericdaugherty.com/dev/itunesexport/">easily export your Playlist data from iTunes</a> is to compensate for a feature that really should exist in iTunes.  At least they made the data relatively easy to access (XML).<br /><br />I also noticed Matt's post didn't touch on Picasa.  Google isn't perfect, and a lot of their existing sharing is for data that has well established formats (Email, Calendar, Docs, etc).  Maybe we need a unified approach for Image tagging/categorization.
